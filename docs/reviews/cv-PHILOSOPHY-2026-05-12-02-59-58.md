## Detailed Reviews

### DeepSeek (deepseek-chat)

Here is the technical review of the provided artifact, "Philosophy."

- **Medium: Ambiguity in "LLMs are probabilistic systems" (Axiom 1)**
    The axiom claims "same input, different outputs" is guaranteed. While technically true for most LLM APIs at default settings (e.g., `temperature > 0`), users can set `temperature = 0` and `top_p = 1` to achieve deterministic output for the same input (though different models or provider versions may still produce different results). The absolute statement could mislead a user into thinking consistency is impossible, when in practice it is partially achievable and often advisable for code generation.
    **Suggested fix**: Clarify that "same input, different outputs" is the common behavior at non-zero temperature, and note that deterministic inference (e.g., temperature = 0, seed fixed) is a pragmatic choice Nexus recommends for code generation tasks.

- **Low: Imprecise definition of "novel architecture design" under "What AI Is Not Effective At"**
    The statement "they remix training data, not invent" is an oversimplification. LLMs can synthesize known architectural patterns into novel combinations that are functionally valid (e.g., combining an attention mechanism with a new loss function). The more precise limitation is that AI cannot reliably validate the correctness of a novel architecture or anticipate emergent issues. This may cause a developer to dismiss a legitimate (if rare) AI-suggested exploration.
    **Suggested fix**: Rephrase to "Novel architecture design — AI can suggest combinations of known patterns, but cannot verify correctness or foresee unintended interactions."

- **Medium: Missing edge case in "When Not to Use Nexus"**
    The document lists "emergency hotfixes (use Hotfix Workflow)" but does not define the Hotfix Workflow anywhere in this artifact. This assumes prior knowledge of an external workflow that is not documented within the project's core philosophy. For a solo developer, this creates ambiguity and potential process drift during emergencies.
    **Suggested fix**: Either (a) add a brief definition inline or as a footnote, or (b) link to the Hotfix Workflow document (e.g., `docs/HOTFIX_WORKFLOW.md`).

- **Low: "Explicit over Implicit" contradicts current artifact structure**
    The artifact itself states that "constraint is not written down, it does not exist." However, it contains several implicit constraints that are not documented: (a) the minimum version of a code editor or AI provider, (b) the expected directory structure or config file format, (c) the exact artifact template names (e.g., "Light Workflow" vs "Hotfix Workflow"). This is a minor maintainability risk — a new user reading only this Philosophy doc cannot reproduce the process without additional, unmentioned files.
    **Suggested fix**: Add a "Prerequisites" or "Related Documentation" section listing required artifacts (e.g., `docs/EXPLORATION.md`, `docs/HOTFIX_WORKFLOW.md`, `docs/LIGHT_WORKFLOW.md`). Even a single line will resolve this.

- **Low: Ambiguous threshold in "When Not to Use Nexus"**
    The phrase "code that will never be read again" is subjective and unverifiable. A solo developer may misjudge this (e.g., label a one-shot script as disposable, only to need it six months later). This could lead to process inconsistency or avoidable tech debt.
    **Suggested fix**: Replace with a concrete heuristic, such as "any file committed to `main` branch with a lifespan expected to exceed 7 days should follow Nexus process." This aligns with the "Git as audit trail" value.

- **Medium: Overly strong claim about "No true reasoning" (Axiom 1)**
    The statement "No true reasoning" is philosophically charged and technically imprecise. Many LLMs demonstrate chain-of-thought reasoning that can be verified and trusted for certain constrained tasks (e.g., simple arithmetic, logic puzzles). The actual risk is that AI *looks like* it reasons but can be wrong, not that it never reasons at all. This could give experienced engineers false confidence that they can completely ignore AI's intermediate steps, even in well-scoped tasks.
    **Suggested fix**: Rephrase to "Reasoning is probabilistic and fallible — AI can follow a chain-of-thought but may skip steps, invent facts, or produce plausible-sounding but incorrect logic. Always verify critical reasoning steps."

- **High: Missing definition of "Light Workflow" and "Hotfix Workflow"**
    As noted above under "Missing edge case," the Philosophy document references two named workflows without defining them. For a solo developer reading this as the entry point, this is a consistency and completeness gap — they cannot apply the philosophy without external knowledge. This error is high severity because it undermines the "Single-author projects" scope: the solo developer should have all process rules in a single logical unit.
    **Suggested fix**: Include a minimal description of each referenced workflow (e.g., one sentence each) within the "When Not to Use Nexus" section, or add a "Workflow Reference" subsection.

- **Low: Decision Principle #5 is ambiguous**
    "If context exceeds what can be explicitly loaded, the scope is too large." This is a good heuristic, but it does not define *how to measure* context or what "explicitly loaded" means concretely. A developer may not know whether they exceed the boundary.
    **Suggested fix**: Add a quantifiable guideline, e.g., "If a single session requires more than 1500 lines of context (spec + code + prompts) to be repeated across multiple prompts, split into multiple phases." (Adjust number based on your average project size.)

- **Medium: Overgeneralization of "AI does not remember prior conversations" (Axiom 2)**
    This is true for stateless APIs (like a single chat session), but many AI tools (Copilot, Claude Projects, custom RAG) support persistent memory via vector stores or file attachments. The statement may mislead a solo developer into ignoring available persistence features that could reduce context load.
    **Suggested fix**: Clarify that "in the context of a single stateless session, AI does not remember before context explicitly provided." Then note that persistent memory/custom instructions are a tool that should be used *with caution* (per the "Explicit over Implicit" value).
